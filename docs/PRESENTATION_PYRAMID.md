# IT Due Diligence Agent - Presentation Pyramid

> **Purpose**: Layered presentation materials for different audiences
> **Date**: January 2026

---

## The Pyramid Structure

```
                    ┌─────────┐
                    │ 3 SLIDES│  <- Investment Committee / Board
                    │   WHY   │     "Why does this matter?"
                    └────┬────┘
                         │
                 ┌───────┴───────┐
                 │   15 SLIDES   │  <- Deal Team / Partners
                 │     WHAT      │     "What does it do?"
                 └───────┬───────┘
                         │
            ┌────────────┴────────────┐
            │        30 SLIDES        │  <- Implementation / Tech Team
            │          HOW            │     "How does it work?"
            └─────────────────────────┘
```

Each layer empowers the next. The 3-slide version pulls from the 15-slide version, which pulls from the 30-slide version.

## Usage Guidelines

### When to Use Each Tier

| Tier | Audience | Context | Goal |
|------|----------|---------|------|
| **3-Slide** | IC, Board, Senior Partners | First exposure, approval meetings | Get buy-in |
| **15-Slide** | Deal Team, Partners | Deal kickoff, training | Enable usage |
| **30-Slide** | Tech Team, Analysts | Implementation, extension | Enable building |

### Presentation Tips

**3-Slide Version:**
- Lead with the problem (everyone knows IT DD is painful)
- Show the output (Cost Center numbers land well)
- End with business case (faster, cheaper, better)

**15-Slide Version:**
- Start with slides 1-3 to orient
- Spend time on Evidence Chain (slide 6) - it's the differentiator
- Demo is critical - show don't tell

**30-Slide Version:**
- Use as reference, not linear presentation
- Jump to relevant sections based on audience questions
- Have code/system open for live demos
